# Project 5 讨论文档：白板编辑器中的「快速创建与连接」交互系统

> 基于你提供的整套设计稿截图整理。目标不是逐页复述设计稿，而是把这些内容收束成一个适合放进作品集的完整项目文档，并明确哪些内容应该做成动态 demo、哪些内容应该作为叙事骨架、哪些内容只适合放到边界 case 折叠区。

## 一、我对这批设计稿的整体判断

这不是 20 多个零散小优化。

它本质上是一个统一的交互系统，主题很明确：

> **用户如何在白板/编辑器中，围绕一个对象进行快速创建、快速连接、快速连续操作，同时又不丢失空间感、可预期性与操作稳定感。**

换句话说，这个项目真正厉害的地方不是某一个单点交互，而是你把下面这些东西做成了一套能互相咬合的系统：

- 创建点的命名与层级
- 热区激活规则
- 不同对象类型的多态反馈
- 点击创建 / 拖拽创建 / 快捷键创建
- 创建后是否自动连接
- 创建目标如何避让碰撞
- 多对象同时出现时连接谁
- 旋转、缩放、视角平移、视角缩放时规则如何保持一致
- 工具条、吸附、文本输入、边界场景如何协同

所以这个项目最适合的包装方式不是“交互拼盘”，而是：

> **一个完整的快速创建与连接系统设计。**

而在作品集里，再把它拆成 3 到 4 个“可演示、可操作、可讲清楚”的交互模块。

## 二、这个项目最适合的最终定位

### 一句话定位

> 我在即时设计的白板/编辑器场景中，设计了一套围绕「创建点」展开的快速创建与连接系统，让用户在高频操作里更快地产生内容、建立关系，并在复杂画布中始终保持空间感与可控感。

- 你能把视觉上很轻的反馈，设计成一套**可预测、可继承、可解释**的模型
- 你对复杂编辑器中的**空间关系、边界条件、优先级判断**非常敏感
- 你具备把“很多细碎规则”收束成一套**用户可以感知到的简单体验**的能力

## 四、适合放进作品集的最终章节结构

## 1. 项目背景

- 场景：即时设计白板 / 编辑器
- 目标：让用户围绕当前对象快速继续创作，而不是频繁回到工具栏重新找入口
- 问题：如果只做一个“创建按钮”，在真实画布里很快会遇到大量复杂问题：
- 该从哪里触发
- 用户怎么知道它能做什么
- 新建对象放在哪里
- 要不要自动连接
- 如果附近有很多对象，应该连谁
- 如果对象旋转了怎么办
- 如果创建位置出屏怎么办
- 如果有碰撞、吸附、文本输入、工具条，又怎么协同

## 2. 核心设计命题

可以用一句话概括：

> **让“围绕对象继续创作”这件事足够快，但又不让系统显得随机。**

我建议在作品集里把你的设计原则明确写出来：

- **可发现**：用户能感知“这里可以继续做下一步”
- **可预测**：创建方向、距离、连接目标不是随机的
- **可连续**：一次操作完成后，用户能顺着上一轮结果继续做
- **可容错**：碰撞、边界、视角、缩放、旋转不会让系统失控
- **可解释**：复杂规则虽然存在，但最终表现要简单

## 3. 系统拆解

建议在这里引出后面的 4 个模块：

- 创建点系统
- 创建模式系统
- 连接判断系统
- 边界稳定性系统

## 五、建议在作品集里呈现的 4 个主模块

## 模块 A：创建点系统

这是整个项目的入口层。

### 这一模块要讲什么

- 名词解释：控制点、连接点、创建点
- 创建点不是永远显示，而是被热区激活
- 热区是相对热区，会随着对象缩放变化
- 部分对象的热区会跟随旋转调整
- 不同对象类型有不同的创建点视觉语言
- 创建点在不同状态下会发生多态变化

### 从设计稿里提炼出的关键信息

- 有明确的三层概念：
- `控制点`
- `连接点`
- `创建点`
- 创建点存在激活逻辑：
- 当画布缩放小于 10% 时，隐藏快速创建点
- 创建点本身是物理像素尺寸，示意图里给到 `14px`
- 热区不是固定死值，而是相对对象计算：
- 跟随对象放大而放大，缩小而缩小
- 是以控制点为基础绘制出来的范围
- 某些规则中存在固定 padding：
- 例如示意里有距离控制点上方 `16px` 固定
- 距离右侧包围盒 padding `0px`
- 旋转对象时，至少有一类热区会跟随对象旋转进行调整
- 不同对象类型的创建点样式不同：
- 图形文本 / 文本 / 图形，更像“箭头型创建”
- 便签，更像“加号型创建”
- 控制点存在多态状态：
- 单选时默认点
- hover 热区时出现 outline 样式
- hover 创建点时出现填充态
- press + drag 时 cursor / preview 会变化
- 预览对象存在 `opacity 30%` 的轻预览状态

### 这个模块最适合的 demo

做一个“创建点实验台”：

- 中间放一个可切换类型的对象：便签 / 图形文本 / 文本 / 图片 / 图形节点
- 右侧面板可以切换：
- 缩放比例
- 是否旋转
- 是否显示热区
- 对象类型
- 访客可以直接 hover，看见：
- 创建点如何被激活
- 热区如何变化
- 为什么某些时候创建点消失
- 不同对象的创建点为什么不是同一个样式

### 这一模块的作品集价值

这能证明你不是“给对象旁边加个按钮”，而是在定义一个**可学习的交互语法**。

## 模块 B：创建模式系统

这是“用户实际怎么创建内容”的主菜。

### 这一模块要讲什么

- 点击创建
- Press + Drag 创建
- 键盘快捷键创建
- 创建后的焦点与输入模式
- 连续创建时如何继承上一轮节奏

### 从设计稿里提炼出的关键信息

#### 1. 点击创建

- 对于便签：
- 按创建点所在方向，在固定位置创建
- 设计稿里多次出现 `64px`
- 上下左右创建都遵循固定方向放置

- 对于图形节点 / 流程节点：
- 距离 hover 方向 `120px` 进行创建
- 用户点击后创建并连接到该节点
- 新建节点默认为原节点类型

#### 2. Press + Drag 创建

- 用户可拖拽创建
- 创建位置取决于鼠标释放点
- `Press 后移动 3px` 或 `Press 超过 0.5s` 进入拖拽创建模式
- 释放后会出现形状选择工具
- hover 时有形状预览
- 适用对象在设计稿中明确写到：
- 便签
- 图形文本
- 文本
- 图片

#### 3. 快捷键创建

- 在选中便签 / 或便利贴输入模式下
- `cmd + enter` 可以创建新的便签
- 新便签遵循碰撞逻辑
- 新便签在层级上位于最顶层
- 设计稿里仍然维持 `64px` 的节奏

#### 4. 创建后的状态流转

- 创建后会聚焦到新节点
- 自动进入输入模式
- 点击空白处关闭相关面板

#### 5. 连续创建逻辑

- 只有在 `Press + Drag` 完成对象创建后，后续继续创建会继承上一轮创建长度
- 如果用户调整了创建源点 A 或新创建 B 的距离，下一次创建会继承调整后的长度

### 这个模块最适合的 demo

做一个“创建方式切换 demo”：

- Demo 内只放一个对象，访客可以：
- 点击创建
- 长按拖拽创建
- 按下快捷键创建
- 页面右侧同步解释：
- 何时进入拖拽创建模式
- 放置位置为何在这里
- 为什么这次会自动进入输入态
- 为什么下一次创建距离会继承

### 这一模块的作品集价值

这部分最能体现你把“多种入口”收成同一套心理模型的能力。

用户看到的是三种方式，背后其实是一套统一规则。

## 模块 C：连接与优先级系统

这是整套设计里最有深度、也最值得做成交互 demo 的部分。

### 这一模块要讲什么

- 创建时何时是“新建”
- 何时是“新建并连接”
- 何时直接连接到已有对象
- 多个候选对象同时出现时优先连接谁
- 连接线从哪里出发、落到哪里
- 旋转对象后，方向判断怎么变

### 从设计稿里提炼出的关键信息

#### 1. 控制点创建图层并建立连接

- 点击创建时，系统可能不是单纯新建对象，而是：
- 新建一个节点
- 并自动建立连接线
- 新建节点默认继承原节点类型

#### 2. 建立连接线热区范围

- 对图形文本与图片类对象：
- 热区以原对象为中心
- 基于“上 / 下 / 左 / 右”衍生
- 面积约为对象 1 倍 + `120px` 固定外延
- hover 创建点时，优先连接热区范围内的对象

- 对白板模式下文本对象：
- 左右热区宽度 = 文本宽度 * 1.5
- 上下热区宽度 = 文本宽度
- 文本为一行时，热区高度 = 文本框高度 * 3
- 两行时 = * 2
- 三行及以上时 = * 1

#### 3. 连接对象优先级

- 多对象同时处于热区范围内时：
- 优先选择距离最近的对象建立连接
- 距离相同时还有 tie-breaker：
- 若创建点方向为左右，优先从上到下建立连接
- 若创建点方向为上下，优先从左到右建立连接

#### 4. 连接线起点与落点

- 连接线起点：
- 按照创建点方位的锚点出发
- 连接线落点：
- 连到目标对象的相反方位

#### 5. 旋转逻辑

- 创建点会根据对象旋转而发生改变
- 具体创建方向规则由创建点所在象限决定
- 在象限边缘情况中，设计稿里甚至给出“默认按角度 +1 处理”的边缘方案

#### 6. 热区与旋转的关系

- 一部分热区会跟随对象旋转
- 另一部分“建立连接热区”是固定方位，不跟随对象一起旋转

这很关键，说明你不是简单地“全都跟着转”，而是按任务目标做了拆分：

- 与对象本体关系更强的规则，跟着转
- 与连接判断稳定性更强相关的规则，保持固定方位

#### 7. 已建立连接后的后续行为

- 当创建连接点方向与被连接对象尚未建立连接时：
- 点击可建立连接

- 当该方向与某被连接对象已经建立连接时：
- 点击会进入“新建节点并建立连接”
- 如果此时热区里已经有节点，则优先连接节点
- 若节点连接线被手动删除，则优先回退到新建节点

#### 8. 删除 / 复制后的连接记忆

- 用户创建节点并连接后，如果手动删除连接线：
- 系统会标记被连接节点
- 下次 hover 创建点时，会优先“添加新节点并建立连接”
- 这类状态刷新网页后不保证保留

- 如果用户复制被标记节点：
- 复制出来的节点标记会被取消
- 但如果复制节点落在连接区域内，hover 创建点仍然可能连接到该节点

### 这个模块最适合的 demo

做一个“连接优先级沙盒”：

- 中央一个源节点
- 周围可拖出多个候选对象
- 支持切换：
- 是否在热区内
- 是否在可视范围内
- 是否已经建立过连接
- 是否手动删除过连接
- 是否复制得到
- 是否旋转
- 页面实时显示系统判断：
- 本次点击会直接连接哪个对象
- 还是新建节点并连接
- 起点 anchor 是哪一侧
- 终点落点在哪一侧

### 这一模块的作品集价值

这是整个项目最能证明“系统判断能力”的部分，也是你和普通交互作品最拉开差距的地方。

## 模块 D：边界、碰撞与视角稳定性

这一块是让系统真正“能落地”的部分。

### 这一模块要讲什么

- 什么对象参与碰撞，什么对象不参与
- 创建时如何避让已有对象
- 出屏之后视角如何平移 / 缩放
- 边界外但热区内时怎么处理
- 吸附打开后如何协同
- 工具条什么时候出现，什么时候隐藏

### 从设计稿里提炼出的关键信息

#### 1. 便签 vs 其他对象的碰撞差异

- 便签：
- 设计稿里明确强调：**便签不和任何节点发生碰撞**

- 图形文本 / 文本 / 图片：
- 不与某些元素碰撞，例如：
- 钢笔
- 高亮笔
- 铅笔
- 切片
- 连接线
- 除此之外，与所有节点（白板模式节点 + 设计模式节点）发生碰撞

#### 2. 碰撞判断与默认距离

- 多处示意图都围绕 `120px` 的默认创建距离展开
- 如果目标区域中寻找到了足够空间：
- 当距离 >= 创建点 + `32`
- 或左右可用空间 >= `16px`
- 则可以在中间居中创建
- 小于阈值则不创建

- 当左右距离相同时：
- 默认向右创建

- 当优先创建位置不在可视区域、另一方向在可视区域时：
- 优先在可视区域内创建

#### 3. 视角平移与缩放

- 当新建节点无法在可视区域显示时：
- 创建后会平移视角

- 平移判断方式：
- 以“创建节点”和“被创建节点”形成有向包围盒
- 让其上下左右的最小 padding >= `64px`

- 如果 padding 仍不足：
- 进行“视角缩放”
- 缩放值以最小 `64px` padding 为目标

- 对于便签单元素创建：
- 更偏向只平移，不做视角缩放

#### 4. 边界与可视范围

- 如果对象完全不在可视区域内，即便位于热区范围，也不建立连接
- 在这种情况下，hover 创建点会回退为：
- 创建新节点并建立连接
- 如新建节点与已有节点相交，再进入碰撞逻辑
- 创建后视角发生平移，并进入输入模式

#### 5. 吸附逻辑

- 开启吸附模式后：
- 新建节点默认吸附到点阵 / 网格
- 若原始节点本身未对齐网格，则后续在该节点上创建的新节点，依然保持与原节点一致的空间逻辑，不会因为后来开了吸附而突然改变节奏

#### 6. 工具条协同

- 控制点与属性工具条之间保持固定距离
- 设计稿中明确出现：
- 工具栏与具备快速创建点的节点，保持 `54px` 间距

- 在拖拽连接线创建 / 拖拽调整连接线时：
- 不展示属性工具条

### 这个模块最适合的 demo

做一个“边界实验室”：

- 让访客把对象拖到画布边缘
- 打开 / 关闭吸附
- 在热区里堆叠多个对象
- 切换非碰撞对象（例如笔迹）
- 切换是否允许对象完全出屏
- 页面直接展示系统当前行为：
- 平移视角
- 缩放视角
- 新建失败
- 居中创建
- 连接已有对象
- 创建并连接新对象

### 这一模块的作品集价值

这一块最能体现你对真实落地复杂度的处理能力。

## 六、从截图里抽出来的“完整内容清单”

如果要做成项目，建议把素材内容收束成以下 8 组知识点。

### 1. 基础概念

- 控制点
- 连接点
- 创建点

### 2. 热区系统

- 激活创建点热区
- 热区相对缩放
- 热区跟随对象旋转
- 文本热区随行数变化

### 3. 控制点视觉状态

- 默认态
- hover 热区态
- hover 创建点态
- press + drag 态
- 不同对象类型的视觉差异（箭头 / 加号）

### 4. 创建模式

- 点击创建
- Press + Drag 创建
- 快捷键创建
- 连续创建
- 继承上一次创建距离

### 5. 连接建立

## 九、哪些内容适合放主线，哪些适合放进折叠区

这是很重要的取舍。

### 适合放主线的内容

- 名词解释：控制点 / 连接点 / 创建点
- 热区激活逻辑
- 点击创建 / 拖拽创建 / 快捷键创建
- 新建并连接
- 多对象时的优先级选择
- 视角平移 / 缩放

### 适合放折叠区的内容

- 文本单行 / 双行 / 三行的热区高度变化
- 已删除连接线后的标记逻辑
- 复制被标记节点后的行为
- 可视区域外但热区范围内对象的特殊判断
- 多对象距离相同情况下的 tie-breaker
- 旋转边界按角度 +1 的处理
- 哪些元素参与碰撞，哪些不参与

也就是说：

**主线讲“体验为什么成立”，折叠区讲“系统如何不出错”。**

## 十一、这个项目最值得强调的设计价值

如果要把这个项目讲高级，我建议不要只讲“我设计了哪些规则”，而是强调这 4 件事：

### 1. 把复杂规则藏在简单体验后面

用户感知到的是顺手，不是规则很多。

### 2. 让创建动作具有空间连续性

创建不是孤立动作，而是沿着空间关系继续生长。

### 3. 在复杂画布中保持可预测性

越是多对象、多方向、多边界条件，越需要稳定的判断逻辑。

### 4. 统一多入口操作体验

点击、拖拽、快捷键看起来不同，但用户不会觉得它们是三套系统。
