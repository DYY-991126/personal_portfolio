## The Scale of the Problem

Building a design system is like building a product for other designers and engineers. Token architecture and component API design were critical.

Global Finance had 47 internal tools and 12 client-facing platforms, each with their own UI patterns. This fragmentation led to inconsistent user experiences and duplicated engineering effort.

## Token Architecture

We built a three-tier token system that scales from brand-level decisions down to component-specific values:

```typescript
const tokens = {
  global: {
    "color-green-500": "#10b981",
    "space-4": "16px",
    "radius-md": "8px",
  },
  semantic: {
    "color-success": "{global.color-green-500}",
    "space-component-gap": "{global.space-4}",
  },
  component: {
    "button-radius": "{semantic.radius-md}",
    "button-padding": "{semantic.space-component-gap}",
  },
};
```

## Results

- **89%** component adoption rate within 6 months
- **60%** reduction in design-to-dev handoff time
- **47 → 1** unified component library replacing fragmented implementations
