## The Data Problem

We needed a way to display millions of data points without overwhelming the user. The existing solution used static charts that couldn't convey temporal patterns or correlations.

## Architecture

The visualization engine was built with a layered rendering approach:

```typescript
interface RenderPipeline {
  dataLayer: WebGLRenderer;
  annotationLayer: SVGRenderer;
  interactionLayer: DOMOverlay;
}

const pipeline: RenderPipeline = {
  dataLayer: new WebGLRenderer({ antialias: true }),
  annotationLayer: new SVGRenderer(),
  interactionLayer: new DOMOverlay({ captureEvents: true }),
};
```

<Callout type="note">
  By separating rendering into three layers, we achieved 60fps with datasets exceeding 2 million data points.
</Callout>

## Key Features

- **Semantic zoom** — Different visualization types appear at different zoom levels
- **Linked brushing** — Selections in one chart propagate across all views
- **Real-time streaming** — WebSocket-based data ingestion with sub-100ms latency
- **Adaptive density** — Automatic aggregation based on viewport and zoom level
