## 背景

**Wegic 是一款面向纯小白的 AI 建站产品。**

产品因为技术架构过时，导致需求推进缓慢、退款率居高不下、业务发展受阻。在 2025 年 9 月，业务进行了组织结构调整，重做这个产品。推出 wegic 2.0

<Project3WegicSitePreview />

## 旧技术架构的弊端

1.0 基于 workflow 架构。用户 query 输入后先用 AI 判断任务类型。再根据结果，特例处理。带来最直观的影响：

1. **退款率高**

AI 没有上下文。用户在修改网页时，直观感受就是 AI “不听话”、“改不成东西”。这是导致退款率居高不下的核心问题

2. **产品日益复杂，逐渐脱离定位**

由于旧架构扩展性差，对话建站产品内的诸多需求被迫采取降级方案，用 GUI 的方式实现。这类 case 的堆积导致产品越来越复杂、难用。逐渐脱离定位。


## 新技术架构带来的变化

2.0 采取的是 Agent 架构。简单来讲，让 AI 变得更聪明，用户的指令能够准确执行。原先诸多用 GUI 使用的交互逻辑，大部分均可用对话交互体系来实现。产品交互方式一致，用户更容易用起来。

## Wegic 2.0 MVP 范围与边界

Wegic 2.0 满足的核心价值以「用户在产品内完成建站为目标」，我们定义了 mvp 的范围与边界。核心做了一件事情，就是砍功能。
1. 用统一的对话交互逻辑平替所有 GUI 特例功能
2. 撇去旧架构下无法移植/不必要的功能

## 界面与交互框架设计

### 界面简化与结构调整

1. 根据范围层梳理结果，移除掉大量 GUI
2. 剩余功能/信息按重要性、频率将核心功能外露至一级界面；次要功能进行收纳二级/概念合并
3. 对话面板调至左侧，加强以「对话为主」印象。并为后续扩展留出空间。

### 改版后优势：

1. 用户任务更清晰，使用更聚焦。从 1.0 录屏观测，用户进入产品后会在各个按钮中来回犹豫、点击探索，整个过程持续 8–15s 左右。
2. 改版后从定性 30 条用户操作录屏来看，用户进入编辑页后平均 5s 内进入对话编辑。

### 交互模式改变

1.0 是默认进入编辑模式，点击网页内的元素出现【对象-功能】面板。这种交互模式带来的弊端是：

1. 突然出现的 GUI 面板增加学习成本
2. 用户容易一开始就陷入细节编辑，但这不是修改网站的最佳路径
3. 用户修改后，如果要预览网页的效果。需要点击进入 Preview 后才可以查看。使用连续性存在问题。
4. 该信息在多个页面/区块内都存在，这种单点修改不如系统性修改

## 对话中的 Agent 运行与任务呈现设计

### 理解 Agent Loop
与传统软件依赖预定义流程不同，Agent 产品里的核心执行机制是 Agent Loop。

Agent 会围绕用户目标，基于当前上下文自主理解任务、调用合适的工具，并根据中间结果持续迭代，直到任务完成或达到停止条件。

### **设计挑战** 
1. 把 Agent Loop 的系统逻辑，转化为**小白用户可理解、可预期的简单体验**
2. 延续 1.0 的三个 IP 形象，**将这三个 IP 利用起来，以 AI Team 的概念进行包装**。让用户感觉一个团队正在为你干活的感觉

### 方案设计思路

#### 基于 Agent 行为设计消息卡片

行为1： **LLM 无须调用工具，文本回复**

行为2： **LLM 调用 Tool 或 Skill 完成任务后回复用户**。对此类行为单独设计了 Agent Looping 卡片。

### Agent Looping 设计

### 设计思路

**明确目标用户**

从 1.0 的市场数据来看，目标用户是无技术背景的小白和小老板。我对小白用户的判断是：他们不需要理解系统逻辑，只需要知道系统是否正在做、做了多少、还剩多少。

**用进度概念取代系统逻辑**

进度条是最直观的预期管理。用户可以一看到正在干活、做到了多少、还剩多少。虽然 Loop 本身无进度概念，但进度条具有安慰剂效应，在一个复杂任务中，可减缓用户等待焦虑。

**用 IP 动画解释 Tools 调用**

我们不需要用户理解 AI 调用工具任务。而是要让用户有一堆人正在帮我干活的感觉。于是，我们将 tools 抽象为了任务概念，并且用人物动画的形式传递。

**信息分层**

在卡片底部，我们只暴露 AI 正在做什么的白话概念。如：“正在整理你的产品信息…”。全量的 Agent Loop 过程，记录在二级面板。

### Agent Looping 设计落地细节

#### 识别系统能力
先梳理清楚系统里有哪些 Tools 和 Skills：

| 类型 | 名称 |

| Tool | assignTaskTool、listDirectoryTool、readFileTool、writeFileTool、searchReplaceTool、genImageTool、editImageTool、genVideoTool、webSearchTool、deepResearchTool、managePlanTool、readLogTool、globTool、grepTool、execCommandTool、skill-enhance、firecrawl |
| Skill | design（website、effect）、visual（hero_video、launch_video、gen_logo） |

将 Tools 和 Skills 按任务概念归类；**每次调用工具时**，卡片上出现使用用户能理解的白话告知用户 AI 正在做什么。

## 项目结果

项目在规定时间内顺利上线。核心目标均已达成：界面框架完成简化，AI 对话流完整适配 Agent 架构。

上线后，Wegic 2.0 的核心业务转化表现显著优于 1.0——**核心转化指标较 1.0 版本提升超过一倍**。在资源有限、时间紧迫的条件下，聚焦核心体验、做出有效取舍，同样可以带来实质性的业务成果。
